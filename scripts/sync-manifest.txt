# sync-manifest — how check-sync.sh reconciles template/.claude vs template/.codex.
#
# The two harness trees share most content but diverge by design in some places
# (file format, provider-specific wiring, provider-only skills). This file
# declares those INTENTIONAL divergences so the drift report stays low-noise:
# it flags only genuine drift, never the permanent by-design differences.
#
# Paths are RELATIVE to each tree root (no .claude/ or .codex/ prefix).
# The LEFT column is always the .claude path; the RIGHT column the .codex path.
#
# Directives:
#   body   <claude-rel> <codex-rel>   compare file bodies (after path normalization).
#                                     Only needed when the relative paths differ;
#                                     same-path files in both trees auto-pair as body.
#   pair   <claude-rel> <codex-rel>   both must exist; format differs, so no body diff.
#   claude <rel-or-prefix/>           intentionally Claude-only; do not flag.
#   codex  <rel-or-prefix/>           intentionally Codex-only; do not flag.
#
# A trailing slash means "directory prefix" (matches everything under it).
# Any same-relative-path file present in BOTH trees and not listed here is
# auto-compared by body. Any file present in only one tree and not allowlisted
# here is reported as structural drift.
